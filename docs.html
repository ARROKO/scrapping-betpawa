Démarrage rapide

Le serveur écoute par défaut sur http://127.0.0.1:3000.
POST/start
Démarre le bot avec les paramètres fournis. Réponse 202 si accepté.
{
  "targetCote": 50,
  "marketId": "DC",
  "autoLogin": "1",
  "stakeAmount": 2,
  "randomMode": "0",
  "maxOddPerSelection": 2.5,
  "selectionMode": "priority",
  "ouLine": 2.5,
  "ouPriority": "over",
  "countryCode": "+237",
  "phoneNumber": "699000000",
  "password": "secret"
}
curl -X POST http://127.0.0.1:3000/start \
  -H "Content-Type: application/json" \
  -d '{
    "targetCote": 50,
    "marketId": "DC",
    "autoLogin": "1",
    "stakeAmount": 2
  }'
GET/status
Retourne l'état courant, dates de début/fin, code de sortie et un extrait d'env.
curl http://127.0.0.1:3000/status
GET/logsSSE
Flux en temps réel des logs via Server-Sent Events.
const es = new EventSource('http://127.0.0.1:3000/logs');
es.onmessage = (e) => {
  const msg = JSON.parse(e.data); // { type: 'log', line: '...' }
  console.log(msg.line);
};

Gestion de session

GET/session
Vérifie la présence d'un profil local et retourne le contenu de .pawa-profile/session.json s'il existe.
curl http://127.0.0.1:3000/session
POST/logout
Supprime la session locale et stoppe le run si actif.
curl -X POST http://127.0.0.1:3000/logout
POST/stop
Demande l'arrêt du processus en cours. 202 si accepté, 409 si aucun run.
curl -X POST http://127.0.0.1:3000/stop

Détails des paramètres (/start)

  • targetCote (number) – Cote totale cible. Alias acceptés: target, target_cote.
  • marketId (string) – Marché: DC (double chance), OU (over/under), etc.
  • autoLogin ("0" | "1") – Active la connexion automatique (.env ou valeurs fournies).
  • stakeAmount (number) – Montant de la mise.
  • randomMode ("0" | "1") – Sélection aléatoire intelligente.
  • randomSkipRate (number) – Taux de saut aléatoire.
  • maxOddPerSelection (number) – Cote maximale par sélection.
  • selectionMode (string) – priority (défaut) ou autre stratégie.
  • ouLine (number) – Ligne pour Over/Under (ex: 2.5).
  • ouPriority (string) – over ou under.
  • countryCode, phoneNumber, password – Infos de connexion (écrase .env si fournis).
Note: le serveur met automatiquement PLACEMENT_AUTO=1 et NON_INTERACTIVE=1 en mode API.

Codes de réponse

  • 202 Accepté – Requête acceptée, traitement en cours.
  • 200 OK – Requête réussie.
  • 409 Conflit – État incompatible (ex: déjà en cours / pas de run).
  • 400 Requête invalide – Corps manquant ou mal formé.
  • 500 Erreur serveur – Erreur interne inattendue.
